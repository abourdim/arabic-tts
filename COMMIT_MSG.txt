feat: initial release — MSA Arabic TTS v1.0.0

Complete production-grade Modern Standard Arabic text-to-speech system
built on VITS + HiFi-GAN with full Arabic NLP preprocessing pipeline.

─────────────────────────────────────────────────────────────────────

WHAT

End-to-end Arabic TTS system targeting Modern Standard Arabic (الفصحى)
only. Takes raw or diacritized Arabic text, runs it through a 6-stage
NLP pipeline, and synthesises 22050 Hz speech via VITS.

─────────────────────────────────────────────────────────────────────

PIPELINE STAGES

  [1] Text Normalization   — numbers, tatweel, abbreviations, hamza
  [2] Diacritization       — CAMeL BERT → Mishkal → passthrough
  [3] G2P Conversion       — 28 consonants, sun-letter assimilation
  [4] VITS Acoustic Model  — Transformer encoder + normalising flow
  [5] HiFi-GAN Vocoder     — embedded in VITS (end-to-end)
  [6] WAV Output           — 22050 Hz, 16-bit

ARABIC NLP

  Normalizer
  • Western (0-9) and Eastern (٠-٩) digit → Arabic word expansion
  • Numbers 0-9999 fully covered (صفر → تسعة آلاف)
  • Tatweel/kashida removal (العـلـم → العلم)
  • Common abbreviation expansion (د. → دكتور، أ. → أستاذ)
  • Hamza form normalisation

  Diacritizer (auto-fallback chain)
  • CAMeL Tools: BERT-based, ~95% DER on MSA — preferred
  • Mishkal: rule-based, ~88% DER — lightweight fallback
  • Passthrough: for pre-diacritized input (auto-detected by ratio)

  G2P
  • All 28 Arabic consonants mapped to IPA-compatible phonemes
  • Pharyngeals: ح→ħ, ع→ʕ
  • Uvulars: خ→x, غ→ɣ, ق→q
  • Emphatics: ص→sˤ, ض→dˤ, ط→tˤ, ظ→ðˤ
  • Glottal stop: ء→ʔ
  • Diacritic vowels: fatha→a, damma→u, kasra→i
  • Tanwin: ً→an, ٌ→un, ٍ→in
  • Sun-letter ال assimilation (الشمس → ashshams)
  • Shadda gemination (bb, ss, ʃʃ…)
  • Prosody pause markers from punctuation (،→short ؛→medium .→long)

─────────────────────────────────────────────────────────────────────

MODEL

  Architecture  : VITS (end-to-end text → waveform)
  Encoder       : Transformer, 6 layers, 192 hidden channels
  Duration pred : Stochastic duration predictor
  Flow          : Normalising flow, 4 coupling layers
  Vocoder       : HiFi-GAN (embedded — no separate stage)
  Sample rate   : 22050 Hz
  Training      : batch=16, lr=2e-4, fp16, segment_size=8192

  Why VITS over alternatives:
  • End-to-end — no inter-stage error accumulation
  • HiFi-GAN built-in — real-time CPU inference
  • Stochastic duration — handles Arabic prosodic complexity
  • Superior naturalness vs FastSpeech 2 / Tacotron 2

API ENDPOINTS

  GET  /health             Server status, model state, device
  GET  /voices             Available voice metadata
  GET  /docs               Swagger UI
  POST /synthesize         Full TTS → base64 WAV
  POST /synthesize/stream  Streaming WAV response
  POST /normalize          Text normalisation only
  POST /diacritize         Add tashkeel to MSA text
  POST /g2p                Grapheme-to-phoneme conversion

  Synthesis parameters:
    text          string  1-2000 chars (required)
    speed         float   0.5-2.0 (default 1.0)
    pitch_shift   float   ±6 semitones (default 0.0)
    noise_scale   float   0.0-1.0 (default 0.667)
    return_phonemes bool  include G2P output (default false)

LAUNCHER (msa_tts_launcher.sh)

  1) Setup       create venv, install deps, download CAMeL data,
                 install VITS, check GPU, verify all
  2) Server      start/stop/restart, Docker, logs, health check,
                 serve frontend, dev mode (--reload)
  3) Training    preprocess dataset, train VITS, resume, TensorBoard,
                 evaluate, ONNX export, list checkpoints
  4) Inference   interactive synthesis, batch synthesis, test each
                 pipeline stage, RTF benchmark, test suite
  5) Config      host, port, workers, device, diacritizer, log level,
                 model path — persisted to .tts_config
  6) Monitor     CPU/RAM, GPU util, API smoke test, log management,
                 Python env info, disk usage
  7) Deploy      Dockerfile, docker-compose, nginx, systemd, gunicorn,
                 push to registry

  Direct commands:
    ./msa_tts_launcher.sh start|stop|restart|status|test|benchmark

EVALUATION TARGETS

  MOS    ≥ 4.0    (human mean opinion score)
  MCD    < 6 dB   (mel cepstral distortion)
  WER    < 5%     (ASR on synthesised audio)
  DER    < 5%     (diacritisation error rate)
  RTF    < 0.3    (real-time factor; < 0.1 for streaming)
  DNSMOS ≥ 3.5    (automatic MOS proxy)
