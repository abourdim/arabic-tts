Why VITS wins for Arabic
VITS combines the acoustic model and HiFi-GAN vocoder into a single end-to-end model, eliminating the error accumulation between separate stages. Its stochastic duration predictor handles Arabic's complex prosodic structure significantly better than autoregressive alternatives.

Tutorial: Diacritization

Arabic written without vowels (undiacritized) is highly ambiguous. The diacritizer adds tashkeel — the short vowel marks — which are essential for correct pronunciation.

Example

Without diacriticsWith diacritics (tashkeel)
كتبكَتَبَ (he wrote) or كُتُبٌ (books)
علمعَلِمَ (he knew) or عِلْمٌ (knowledge)

Backend priority

1
CAMeL Tools (BERT-based)
~95% DER accuracy. Uses a fine-tuned BERT model on MSA corpora. Requires camel_data -i defaults download (~2 GB).
2
Mishkal (rule-based)
~88% DER. Lightweight, no download needed. Fast inference. Good fallback for production environments with limited storage.
3
Passthrough
No diacritization. Used when input is already fully diacritized (auto-detected by ratio check), or when neither NLP library is installed.

API usage

bash
curl -X POST http://localhost:8000/diacritize \
  -H "Content-Type: application/json" \
  -d '{"text":"الكتاب مفيد"}'

Tutorial: G2P Conversion

Grapheme-to-phoneme converts diacritized Arabic text into a phoneme sequence that the neural TTS model can process.

Glottalء أ إʔأَمَل، سُؤَال
Fricativesث ذ شth ð ʃثَلَج، ذَهَب

Sun-letter assimilation

When the definite article ال precedes a sun letter, the ل assimilates:

text
Input:   الشَّمْسُ  (the sun)
Written: al-shamsu
Spoken:  ash-shamsu  ← ل assimilates to ش

Input:   الْكِتَابُ  (the book)
Written: al-kitabu
Spoken:  al-kitabu   ← ك is moon letter, no change

API usage

bash
curl -X POST http://localhost:8000/g2p \
  -H "Content-Type: application/json" \
  -d '{"text":"مَرْحَبًا"}'

# Returns: "m a r ħ a b an aː"

Tutorial: Speech Synthesis

Basic synthesis

python
import requests, base64

response = requests.post(
    "http://localhost:8000/synthesize",
    json={
        "text": "السَّلَامُ عَلَيْكُمْ",
        "speed": 1.0,
        "pitch_shift": 0.0,
        "noise_scale": 0.667,
        "return_phonemes": True
    }
)

data = response.json()
wav = base64.b64decode(data["audio_b64"])

with open("output.wav", "wb") as f:
    f.write(wav)

print(f"Duration: {data['duration_s']}s")
print(f"RTF: {data['rtf']}")
print(f"Phonemes: {data['phonemes']}")

Streaming synthesis

python
import requests

with requests.post(
    "http://localhost:8000/synthesize/stream",
    json={"text": "مرحبا بالعالم", "speed": 1.0},
    stream=True
) as r:
    with open("output.wav", "wb") as f:
        for chunk in r.iter_content(4096):
            f.write(chunk)

Tutorial: Training Your Own Model

Requirements for training
Training VITS requires a CUDA-capable GPU (8 GB+ VRAM), at least 10 hours of MSA audio, and takes 3–7 days on a single RTX 3080.
1
Acquire MSA dataset
Download Arabic Speech Corpus (ASC) from en.arabicspeechcorpus.com. This is 1,813 sentences, fully diacritized, studio quality — the best starting point for MSA TTS.
2
Preprocess via launcher
Use Launcher → Training → Prepare dataset. It resamples audio to 22050 Hz and creates train/val split files.
bash
./msa_tts_launcher.sh  # → 3 Training → 1 Prepare dataset
3
Generate training config
Launcher → Training → Generate config creates configs/arabic_msa_vits.json with all hyperparameters pre-tuned for Arabic MSA.
4
Start training
bash
# Via launcher (recommended — handles all setup):
./msa_tts_launcher.sh  # → 3 → 2 Start training

# Or direct:
cd vits
python train.py \
  --config ../configs/arabic_msa_vits.json \
  --model_dir ../checkpoints
5
Monitor with TensorBoard
bash
./msa_tts_launcher.sh  # → 3 → 4 TensorBoard
# Visit http://localhost:6006
6
Load checkpoint into server
bash
TTS_MODEL_PATH=checkpoints/G_latest.pth \
./msa_tts_launcher.sh start

MSA purity matters
Mixing dialectal Arabic (Egyptian, Gulf, Levantine) into the training data will corrupt the model's prosody and phoneme distributions for MSA output. Always verify recordings with an MSA-native speaker.

FAQ

Yes. Without a checkpoint, the system runs in demo mode — it generates a formant-like synthesized tone proportional to the input text length. This lets you test the full API, frontend, normalization, diacritization, and G2P pipeline without any model. The frontend also has a built-in Web Audio API fallback that works completely offline.
The system is designed and trained exclusively for Modern Standard Arabic (الفصحى). Dialectal input will be processed but the output quality will be poor because the phoneme mappings, diacritizer, and acoustic model are all calibrated for MSA phonology and prosody. For dialect support, a separate model trained on dialect-specific data would be needed.
On a single NVIDIA RTX 3080 (10 GB VRAM), training VITS on ~10 hours of MSA audio takes approximately 3–5 days to reach production quality (~400k steps). On an A100 80 GB, this reduces to ~18–24 hours. The model starts producing intelligible speech around 50k steps (~6 hours on RTX 3080).
CAMeL Tools uses a fine-tuned BERT model for diacritization, achieving ~95% Diacritization Error Rate accuracy on MSA text. It requires downloading ~2 GB of model data.

Mishkal is rule-based and achieves ~88% DER. It's much lighter, has no download requirement, and is significantly faster. For most TTS use cases where absolute phonemic accuracy isn't critical, Mishkal is a good production choice.

The system auto-detects which is available and falls back gracefully.

RTF on a modern CPU (Intel i7/i9, Apple M2+) is typically 0.3–0.8, which is acceptable for non-streaming use cases. Training on CPU is not recommended — it would take weeks instead of days.

The frontend demo already handles the Web Audio API synthesis fallback. Troubleshooting

Server won't start

bash
# Check logs
./msa_tts_launcher.sh logs

# Common fix: port already in use
lsof -i :8000
kill -9 <PID>

# Or change port in config:
./msa_tts_launcher.sh  # → 5 Config → 2 Set port

CAMeL Tools import error

bash
# Reinstall with correct version
pip install camel-tools==1.5.2
camel_data -i defaults

# If data is corrupt:
rm -rf ~/.camel_tools
camel_data -i defaults

CUDA out of memory during training

Fix: reduce batch size
In configs/arabic_msa_vits.json, reduce "batch_size" from 16 to 8 or 4. Also reduce "segment_size" from 8192 to 4096 for lower VRAM usage.

Frontend shows "API: offline"

This is expected when the backend isn't running. The frontend works in demo mode regardless — you can still synthesize audio using the Web Audio API fallback. To connect your backend, update const API_BASE in the HTML file and start the server.

Arabic text appears left-to-right

Ensure your browser handles RTL text properly. The textarea has dir="rtl" set. Some OS paste operations strip directional markers — type directly in the textarea or use the quick-phrase buttons.
